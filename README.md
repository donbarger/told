# Told

**The parables, firsthand.**

Told is an interactive parables project by IMB Innovation. Nineteen of the stories Jesus told, stepped into one choice at a time. You're not watching from a pew — you're the younger son, the Samaritan, the rich ruler, the widow with two coins. Each scene unfolds as a card of prose; a choice determines the next scene. Fixed story beats are authored; the scenes between are generated by AI in response to where you've been. The arc is fixed. The texture of your journey is your own.

**Live at [told.thegreatpursuit.faith](https://told.thegreatpursuit.faith)** (IMB Innovation droplet, Caddy + Let's Encrypt, PM2).

## Stack

- **Frontend** — React 18 + Vite, vanilla CSS on the IMB brand system (Nunito Sans display, Source Sans 3 body, Lora for narrative serif, IMB navy / teal / red / orange palette on a light surface)
- **Backend** — Node + Express, SQLite (`better-sqlite3`), Passport (Google OAuth 2.0), JWT sessions, express-rate-limit
- **AI scene generation** — OpenRouter (provider-agnostic proxy) calling Anthropic Claude
- **Narration** (optional) — self-hosted Kokoro-compatible TTS endpoint with on-disk WAV cache
- **Production** — Digital Ocean droplet, Caddy reverse proxy with auto-TLS, PM2 process manager

## Three presentation templates

Each story declares a `template` (default `witness`) and the app dispatches the UI accordingly:

- **`witness`** *(default)* — contemplative full-page reader. No stat bars during play. Two reflection-style buttons. Paper-warm background.
- **`encounter`** — full-bleed single-scene reader with two tap-buttons. Intended for short single-encounter parables. No stats.
- **`journey`** — the original card-swipe deck with a vertical phase-timeline journal sidebar. Good for multi-phase journey stories.

All three render the same AI pipeline output — only the shell differs.

## Architecture

```
frontend/
  src/
    App.jsx                  # routes by screen + story.template
    hooks/
      useAuth.js             # Google OAuth flow + JWT storage + /me prefs
      useGame.js             # story-agnostic game engine (stat1/stat2/stat3)
    stories/                 # one file per parable (card data, stat config, phase map, AI phase rules)
    components/
      LandingScreen.jsx      # pre-auth: navy hero + sign in + featured illustrations
      StorySelectScreen.jsx  # post-auth: hero recommendation + grid + "played" badges
      TitleScreen.jsx        # story intro — scripture + begin
      GameScreen.jsx         # 'journey' template (swipe card + timeline sidebar)
      EncounterScreen.jsx    # 'encounter' template
      WitnessScreen.jsx      # 'witness' template (current default)
      EndScreen.jsx          # final stats, passage, journal by phase, Get-Involved CTA
      AdminScreen.jsx        # aggregate stats + user table + block/unblock
      SettingsModal.jsx      # account + autoplay toggle + admin link
      StoryArt.jsx           # inline SVG illustrations per story

backend/
  server.js                  # binds 127.0.0.1:PORT (Caddy fronts it)
  lib/
    auth.js                  # JWT sign/verify + optionalAuth + requireAuth + requireAdmin (ADMIN_EMAILS gate)
    db.js                    # SQLite schema + auto-migrations
    prompts/                 # one system prompt per story + shared scaffold
  routes/
    auth.js                  # /google, /google/callback, /me, /me/prefs
    game.js                  # /start, /complete, /next (AI scene generation)
    admin.js                 # /users, /stats, /users/:id/{block,unblock}
    tts.js                   # proxy + SHA-256 disk cache of narration audio
  data/                      # users.db (gitignored, 600 perms in prod)
  cache/tts/                 # WAV cache (gitignored)
```

## Run it locally

### Prerequisites

- Node 20+
- A Google OAuth 2.0 Client with `http://localhost:3002/api/auth/google/callback` added to **Authorized redirect URIs** and `http://localhost:5175` to **Authorized JavaScript origins**
- An OpenRouter API key

### Setup

```bash
# Backend
cd backend
cp .env.example .env
# Fill in OPENROUTER_API_KEY, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, JWT_SECRET, ADMIN_EMAILS
npm install
npm run dev                   # 127.0.0.1:3002

# Frontend (second terminal)
cd frontend
npm install
npm run dev                   # 127.0.0.1:5175
```

Visit [http://localhost:5175](http://localhost:5175).

### Environment variables (`backend/.env`)

| Key | Purpose |
|---|---|
| `PORT` | Backend port (default 3002) — always binds `127.0.0.1` |
| `FRONTEND_URL` | Origin of the frontend (dev: `http://localhost:5175`; prod: `https://told.thegreatpursuit.faith`) — used for CORS + OAuth redirect |
| `BACKEND_URL` | Where Google redirects after OAuth — must match the registered callback |
| `JWT_SECRET` | Signing key for session tokens. Use `openssl rand -base64 48` |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | OAuth 2.0 credentials |
| `OPENROUTER_API_KEY` | Scene generation (Anthropic via OpenRouter) |
| `ADMIN_EMAILS` | Comma-separated admin emails (default `dbarger@imb.org,donbarger@gmail.com`) |
| `TTS_SERVER_URL` *(optional)* | Kokoro-compatible narration endpoint |
| `HF_TTS_VOICE` *(optional)* | Voice key (default `am_michael`) |

## Admin

Admins (emails in `ADMIN_EMAILS`) get a panel via **Settings → Open admin panel**:

- Player count, games started / completed, completion rate, average acts per run
- Per-user table with last-seen time, games played, and a block/unblock action

Google OAuth handles authentication — no passwords to reset. Blocking a user cuts off access on their next `/me` poll.

## Security notes

- Backend binds `127.0.0.1` only — Caddy (in front) terminates TLS and applies CSP, HSTS, X-Frame-Options, Referrer-Policy, Permissions-Policy. Hitting the port directly from outside the box is impossible.
- `.env` lives at `600 root:root` on the droplet. It is `.gitignored`. Secrets have never been committed to git (verify: `git log --all -p -- backend/.env`).
- `users.db` is `600` in production. Client-visible JWTs carry only `{sub}` (the user ID); emails/admin flags resolve server-side on every request.
- `/api/game/*` and `/api/admin/*` are behind `requireAuth` / `requireAdmin`. Blocked users get a `403 {blocked: true}` on `/me`.
- Rate limit: 60 requests / minute / IP applied to all `/api/*`.
